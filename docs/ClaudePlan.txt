
     QuantiForge MVP — Implementation Plan
  
     Context

     QuantiForge is a greenfield academic capstone project: a low-code deep-learning simulation platform that lets a student
     visually configure, train, and compare time-series neural network architectures against ARIMA on hourly US equity data. The
     presentation target is a polished Windows desktop demo on a projected screen.                            

     The architecture is fully defined across five spec documents in /docs. The implementation starts from an empty directory.
     The goal of this plan is to build the MVP in logical, testable milestones that progressively assemble the full system.

     Source-of-truth precedence (per CLAUDE.md): DataEngineering_Specs > SystemArchitecture_Specs > PRD > UI_UX_Specs.

     Phase 2 — Python Daemon: Data Pipeline

     Goal: Full data ingestion → gap handling → split → denoising → scaling → windowing pipeline. All logic is in daemon/data/.

     2.1 — Data Ingestion (data/ingestion.py)

     fetch_ohlcv(ticker: str, period: str = "2y", interval: str = "1h") -> pd.DataFrame

     - Uses yfinance.download(ticker, period=period, interval=interval, auto_adjust=False) (unadjusted)
     - Returns DataFrame with columns: open, high, low, close, volume, timestamp
     - Applies gap handling rules (DataEngineering_Specs §2.3) in order:
       a. Filter to 09:30–16:00 ET Monday–Friday
       b. Drop trading days with fewer than 6 regular-session bars
       c. Forward-fill isolated single missing bars (max 1 bar); drop full day if gap > 1 bar
       d. Drop rows with non-positive prices or negative volume
       e. Reset integer index; add continuous boolean flag: True for every row that follows the previous row with no invalid gap.
     The only rows marked False are the first row in the entire DataFrame, and any row that follows a dropped partial session or
     an unfilled multi-bar provider gap. Normal overnight/weekend gaps between clean consecutive trading days are not marked
     discontinuous — the gap is already eliminated by step 1 (market-hours filter), so consecutive clean rows should be treated
     as continuous for windowing purposes.
     - Returns clean DataFrame + metadata dict {ticker, row_count, date_range}

     Cache fetched DataFrame in daemon process memory keyed by ticker (invalidated after 1 hour or daemon restart).

     2.2 — Train/Val/Test Split (data/ingestion.py)

     chronological_split(df: pd.DataFrame) -> tuple[pd.DataFrame, pd.DataFrame, pd.DataFrame]

     - 80/10/10 by row count, never shuffle
     - Returns (train_df, val_df, test_df)

     2.3 — Denoising (data/denoising.py)

     denoise(train_df, val_df, test_df, method: str) -> tuple[pd.DataFrame, pd.DataFrame, pd.DataFrame, dict]

     method is "None", "Kalman", or "DWT". Returns the three DataFrames with close replaced, plus a denoiser_params dict for
     persistence.

     - None: identity pass-through; denoiser_params = {}
     - Kalman: pykalman.KalmanFilter with transition_matrices=[[1]], observation_matrices=[[1]],
     initial_state_mean=train_close[0]; fit (em()) on train split only; apply filter() transform to val and test
     - DWT: pywt.wavedec(train_close, 'db4', level=2); compute σ from MAD of cD1; apply universal threshold; pywt.waverec; trim
     to original length. Reuse same σ for val/test. Store sigma in denoiser_params.

     2.4 — Scaling (data/scaling.py)

     fit_scalers(train_df) -> dict[str, MinMaxScaler]
     scale_split(df, scalers) -> np.ndarray
     inverse_transform_close(values, scalers) -> np.ndarray

     - One MinMaxScaler(feature_range=(-1, 1)) per column: open, high, low, close, volume
     - Fit on train only (fit_transform); transform-only for val/test/live
     - joblib.dump to ./models/{run_id}_scaler.joblib

     2.5 — Windowing (data/windowing.py)

     build_windows(scaled_array, sequence_length: int, continuity_flags: list[bool]) -> tuple[np.ndarray, np.ndarray]

     - Returns (X, Y) with shapes (N, L, 5) and (N, 1)
     - Respects continuity_flags: a window starting at index i is valid only if continuity_flags[i+1 .. i+L] are all True (i.e.,
     no dropped partial sessions or unfilled gaps within the window). Normal overnight/weekend gaps produce no discontinuity
     marker, so windows freely span day boundaries — that is the correct behaviour for a 24/48/72-bar hourly lookback.
     - Returns PyTorch TensorDataset for use with DataLoader

     Verification: Run the full pipeline on SPY; assert train shape ≈ (2400, L, 5) and test shape ≈ (300, L, 5). Verify scaler
     round-trip: inverse_transform(scale(close)) ≈ close.

     Phase 3 — Python Daemon: ML Engines

     Goal: All five architectures + training loop + metrics + artifact saving. ARIMA baseline.

     3.1 — Architecture Implementations (training/architectures.py)

     Each is a nn.Module taking (batch, seq_len, n_features=5) input, outputting (batch, 1).

     ┌─────────────┬────────────────────────────────────────┬────────────────────────────────────┐
     │    Class    │              Key Modules               │               Notes                │
     ├─────────────┼────────────────────────────────────────┼────────────────────────────────────┤
     │ LSTMModel   │ nn.LSTM, nn.Dropout, nn.Linear         │ hidden_size from config            │
     ├─────────────┼────────────────────────────────────────┼────────────────────────────────────┤
     │ GRUModel    │ nn.GRU, nn.Dropout, nn.Linear          │ Same pattern as LSTM               │
     ├─────────────┼────────────────────────────────────────┼────────────────────────────────────┤
     │ BiLSTMModel │ nn.LSTM(bidirectional=True), nn.Linear │ Linear input dim = hidden_size * 2 │
     ├─────────────┼────────────────────────────────────────┼────────────────────────────────────┤
     │ TCNModel    │ nn.Conv1d, nn.ReLU, nn.LSTM, nn.Linear │ Conv1d extracts local feature maps │
     └─────────────┴────────────────────────────────────────┴────────────────────────────────────┘

     build_model(config: dict) -> nn.Module — factory function routing on config["backbone"].

     3.2 — TFT Path (training/tft_path.py)

     Activated when backbone == "TFT". Does not use the generic tensor pipeline.

     1. After denoising + scaling, reconstruct a DataFrame from the scaled train+val arrays (rows in chronological order)
     2. Add two synthetic columns required by TimeSeriesDataSet:
       - time_idx: integer range(len(df))
       - ticker_id: constant string "0" (group identifier; TFT needs at least one group)
     3. Construct TimeSeriesDataSet:
     TimeSeriesDataSet(
         df,
         time_idx="time_idx",
         target="close",
         group_ids=["ticker_id"],
         max_encoder_length=sequence_length,   # L from config
         max_prediction_length=1,
         time_varying_unknown_reals=["open", "high", "low", "close", "volume"],
         target_normalizer=None,               # already MinMax-scaled
     )
     4. Split into train/val TimeSeriesDataSets using training_cutoff = int(0.8 * len(df)); val set uses the same dataset with
     predict=True
     5. Create TemporalFusionTransformer.from_dataset(training_dataset, learning_rate=lr, ...)
     6. Train with pytorch_lightning.Trainer(max_epochs=epochs, callbacks=[...]). Emit EPOCH_METRIC by subclassing Callback and
     overriding on_train_epoch_end; call back into the async WS sender via asyncio.run_coroutine_threadsafe.
     7. Save model.state_dict() on completion; denoiser_params saved as usual

     Errors during TimeSeriesDataSet construction or trainer.fit must be caught and emitted as TRAINING_FAILED.

     3.3 — Training Loop (training/trainer.py)

     async def run_training(ws, run_id, config, device) — called from the WebSocket handler.

     Generic path (LSTM/GRU/BiLSTM/TCN):
     1. Call data pipeline (ingest → split → denoise → scale → window)
     2. Build model; move to device; set optimizer (AdamW/SGD via torch.optim; Ranger via ranger-adabelief or torch_optimizer)
     3. MSELoss training loop; emit EPOCH_METRIC JSON per epoch via await ws.send_json(...)
     4. On completion: compute test metrics (Phase 3.4); save artifact bundle; write registry record; emit TRAINING_COMPLETE
     5. Catch torch.cuda.OutOfMemoryError → emit TRAINING_FAILED with descriptive message

     Handle STOP_TRAINING message by setting a cancellation asyncio.Event.

     3.4 — Metrics (training/metrics.py)

     All computed on inverse-transformed USD values from the test split.

     compute_metrics(y_true: np.ndarray, y_pred: np.ndarray) -> dict

     Returns {"rmse": float, "mae": float, "mape": float, "directional_accuracy": float} using the exact formulas from
     DataEngineering_Specs §8.

     3.5 — Artifact Saving

     On TRAINING_COMPLETE, save the full five-file bundle:
     torch.save(model.state_dict(), f"./models/{run_id}.pt")
     joblib.dump(scalers, f"./models/{run_id}_scaler.joblib")
     json.dump(config, open(f"./models/{run_id}_config.json", "w"))
     json.dump(metrics, open(f"./models/{run_id}_metrics.json", "w"))
     json.dump(denoiser_params, open(f"./models/{run_id}_denoiser.json", "w"))  # {} for None

     denoiser_params must be saved so the inference engine can re-apply the correct denoising transform on live bars. For Kalman,
     this includes fitted Q/R covariances and the final filter state vector. For DWT, this is the training-fit σ threshold.

     Then db.upsert_run(run_id, ticker, backbone, denoiser, hyperparams, metrics, weights_path, status="completed").

     3.6 — ARIMA Baseline (arima/baseline.py)

     run_arima(ticker: str) -> dict

     1. Fetch + preprocess data (no denoising, no scaling — ARIMA operates on raw close USD)
     2. Fit ARIMA(5,1,0) on training close prices using statsmodels.tsa.arima.model.ARIMA
     3. Generate out-of-sample walk-forward forecasts over the test horizon using the expanding-window approach:
     history = list(train_close)
     predictions = []
     for actual in test_close:
         model = ARIMA(history, order=(5, 1, 0))
         fit = model.fit()
         predictions.append(fit.forecast(1)[0])
         history.append(actual)   # expand window by one true observation
     This avoids in-sample leakage. Do not use fit.predict() on the test date range with the training-only model.

     Runtime constraint: For ~300 test steps at order (5,1,0), a naïve per-step refit can take 60–120 seconds. To stay under 30
     seconds, use statsmodels's append/online-update API instead of refitting from scratch each step:
     result = ARIMA(train_close, order=(5,1,0)).fit()
     for actual in test_close:
         predictions.append(result.forecast(1)[0])
         result = result.append([actual], refit=False)   # O(1) update
     refit=False updates the filter state without re-estimating parameters — valid for this fixed-order MVP baseline. Test the
     actual runtime early (Phase 3 verification) and fall back to the naïve refit only if refit=False results look unreasonable.

     4. Compute the same four metrics against test-set actuals (identity inverse-transform since values are already USD)
     5. Return {ticker, order, metrics, predictions}

     Expected runtime < 30 seconds with refit=False.

     Verification: Train an LSTM on SPY with 5 epochs; assert EPOCH_METRIC events arrive via WebSocket; assert artifacts exist in
     ./models/; assert registry record in SQLite. Run ARIMA; assert metrics dict is populated.

     Phase 4 — Python Daemon: Inference Engine & Full Registry API

     Goal: Live inference with rolling buffer, complete CRUD registry endpoints.

     4.1 — Inference Engine (inference/engine.py)

     class InferenceEngine:

     Each active run_id entry holds:
     {
       "model":          nn.Module,          # loaded state_dict
       "scalers":        dict[str, Scaler],  # per-feature MinMaxScaler
       "config":         dict,               # backbone, sequence_length, denoiser
       "denoiser_params": dict,              # training-fit Q/R or σ; {} for None
       "raw_close_buf":  deque(maxlen=L),    # raw USD close values (for DWT / Kalman)
       "scaled_buf":     deque(maxlen=L),   # each element is one scaled OHLCV row (shape (5,))
       "kalman_state":   tuple | None,       # (x_hat, P) — carries Kalman filter state forward
     }

     load_run(run_id) — loads {run_id}.pt, {run_id}_scaler.joblib, {run_id}_denoiser.json, and {run_id}_config.json. Pre-seeds
     raw_close_buf and scaled_buf with the last L bars of the historical dataset for that ticker.

     infer(run_id, bar: dict) -> float — per-bar inference pipeline:
     1. Extract raw OHLCV from bar
     2. Denoising on close (using training-fit params):
       - None: use raw close directly
       - Kalman: apply one Kalman filter step using saved (Q, R) and the carried kalman_state; update state; use filtered close
       - DWT: append raw close to raw_close_buf; when buffer is full apply DWT denoising over the entire buffer using the
     training-fit σ threshold; use the last denoised value. DWT is not meaningful on a single bar — it operates on the rolling
     close history of length L.
     3. Apply scalers.transform([ohlcv_row]) using training-fit scaler
     4. Append scaled row to scaled_buf
     5. When scaled_buf is full (length == L): stack into (1, L, 5) tensor; call model.forward() with torch.no_grad();
     inverse-transform predicted close → USD; return predicted_close
     6. If buffer not yet full: return None (inference deferred until buffer warms up)

     Multiple run_ids can have active entries simultaneously.

     4.2 — WebSocket INFER Handler

     In the main WebSocket message handler:
     case "INFER":
         result = engine.infer(run_id, bar)
         if result is not None:            # None means buffer is still warming up — do not emit
             await ws.send_json({
                 "event": "INFER_RESULT",
                 "run_id": run_id,
                 "timestamp": next_hour_timestamp,
                 "predicted_close": result
             })
         # (buffers are pre-seeded at load_run time, so this guard is rarely triggered
         # in practice, but must be present for correctness on the first few live bars)

     Phase 9 — Next.js: Live Stream & Simulation Mode

     Goal: Alpaca WebSocket live inference and setInterval simulation replay.

     9.1 — Alpaca WebSocket (lib/alpacaSocket.ts)

     Singleton module (mirrors daemonSocket.ts pattern):
     - connect(key, secret, ticker) — opens Alpaca Markets Free Tier WebSocket
     - Subscribes to {ticker} bars. Prefer the 1Hour bar subscription if the Alpaca free-tier WebSocket supports it (use
     {"action":"subscribe","bars":["AAPL"]} with timeframe set to 1Hour in the data stream URL). If only 1-minute bars are
     available on free tier, accumulate 60 one-minute bars then aggregate to a single hourly OHLCV bar (take the first open, the
     max high, the min low, the last close, and the sum of volume across the 60 bars) before forwarding to the INFER path.
     - On each complete hourly bar: calls dispatchInferForActiveOverlays(bar) — iterates useRegistryStore.activeChartOverlays and
     sends INFER to daemon for each
     - Auto-reconnect: 3 attempts, exponential backoff; toast notification on disconnect
     - Called when isLiveModeActive becomes true in useSystemStore; disconnected when false

     9.2 — Simulation Mode (lib/simulationRunner.ts)

     - start(ticker, speed):
       a. Load fetch('/sim_data/{ticker}_sim.json') — array of OHLCV bar objects
       b. setInterval(() => dispatchInferForActiveOverlays(nextBar), intervalMs) where intervalMs = (3600 / speed) * 1000
       c. On last bar: stop interval; toast "Simulation complete"
     - stop() — clears interval
     - setSpeed(speed) — clears and restarts interval at new speed
     - Called when isSimulationActive becomes true; stopped when false

     Speed multiplier: 1× = 3600s (real-time), 10× = 360s, 30× = 120s, 60× = 60s per simulated hour.

     9.3 — dispatchInferForActiveOverlays(bar)

     Shared helper called by both live and sim paths:
     for (const runId of useRegistryStore.getState().activeChartOverlays) {
       daemonSocket.send({ action: "INFER", run_id: runId, bar })
     }

     The daemon INFER_RESULT flows back through daemonSocket.onmessage → chartBridge.appendInferencePoint() →
     chart.update('none').

     Verification: Run simulation with 2 models toggled; verify inference lines append to chart in real time at 60× speed; verify
     SIMULATION banner visible; speed control changes replay rate.

     11.2 — RunPod Polling (Frontend)

     In useTrainingStore or a dedicated hook:

     - On FAILED: useTrainingStore.handleTrainingFailed() with RunPod error message
